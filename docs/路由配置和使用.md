# 路由配置和使用

本文说明项目中路由配置的结构含义、运行时转换流程，以及后续新增页面路由时需要同步修改的位置。

## 整体链路

项目路由以 `src/router/mockRoutes.ts` 作为配置源，配置项类型定义在 `src/router/types.ts`。

运行时主要经过两条链路：

1. 页面路由链路：`mockRoutes` 经过 `transformRoutes` 转换为 React Router 可识别的路由对象，再由 `createBrowserRouter` 挂到 `MainLayout` 下。
2. 菜单链路：`mockRoutes` 经过 `createMenuItems` 转换为 Ant Design `Menu` 的 `items`，用于桌面端侧边栏和移动端抽屉菜单。

相关文件：

- `src/router/mockRoutes.ts`：模拟后端返回的原始路由配置。
- `src/router/types.ts`：路由配置、路由元信息和应用内部路由对象类型。
- `src/router/transform.tsx`：把原始配置转换成 React Router 路由对象。
- `src/router/menu.tsx`：把原始配置转换成 Ant Design 菜单数据。
- `src/router/routeModules.ts`：通过 `import.meta.glob` 收集 `src/pages/**/index.tsx` 页面模块。
- `src/router/index.tsx`：创建应用最终使用的 `router`。

## 根路由结构

`src/router/index.tsx` 中使用 `createBrowserRouter` 创建根路由：

```tsx
const router = createBrowserRouter([
  {
    path: "/",
    element: (
      <Suspense fallback={<div className="p-4">加载中...</div>}>
        <MainLayout />
      </Suspense>
    ),
    children: [
      {
        index: true,
        element: <Navigate to="/home" replace />,
      },
      ...layoutRoutes,
      { path: "*", element: <NotFound /> },
    ],
  },
]);
```

含义如下：

- `/` 使用 `MainLayout` 作为整体布局，布局内部通过 `<Outlet />` 渲染子页面。
- 访问 `/` 时会重定向到 `/home`。
- `layoutRoutes` 来自 `transformRoutes(mockRoutes)`。
- 未匹配到的路径会进入 `NotFound` 页面。

因此，新增普通业务页面时，通常只需要新增页面组件和修改 `mockRoutes.ts`，不需要手动改 `src/router/index.tsx`。

## 路由配置字段

`mockRoutes` 的每一项都是一个 `BackendRouteItem`：

```ts
export type BackendRouteItem = {
  path: string;
  name: string;
  component?: string;
  redirect?: string;
  meta?: BackendRouteMeta;
  children?: BackendRouteItem[];
};
```

### path

路由路径片段。子路由的 `path` 使用相对路径，不要以 `/` 开头。

示例：

```ts
{
  path: "getting-started",
  children: [
    {
      path: "hello-world",
    },
  ],
}
```

上面的子页面最终路径是 `/getting-started/hello-world`。

### name

路由名称，主要用于标识和调试。建议使用稳定、语义清晰的名称，例如 `HelloWorld`、`ResolutionScale`。

### component

页面组件路径，对应 `src/pages/${component}/index.tsx`。

示例：

```ts
{
  path: "hello-world",
  component: "getting-started/hello-world",
}
```

对应页面文件：

```text
src/pages/getting-started/hello-world/index.tsx
```

`routeModules.ts` 使用如下方式自动收集页面模块：

```ts
export const pageModules = import.meta.glob<PageModule>(
  "../pages/**/index.tsx",
);
```

因此页面文件需要放在 `src/pages` 下，并使用 `index.tsx` 作为入口文件。

### redirect

重定向目标。常用于只有子路由、没有实际页面内容的父级路由。

示例：

```ts
{
  path: "getting-started",
  redirect: "/getting-started/hello-world",
  children: [
    {
      path: "hello-world",
      component: "getting-started/hello-world",
    },
  ],
}
```

访问 `/getting-started` 时，会重定向到 `/getting-started/hello-world`。

### children

子路由数组，支持多级嵌套。父路由如果没有 `component` 但有 `children`，转换后会使用 `<Outlet />` 作为父路由元素，让子页面有位置可以渲染。

### meta

路由业务元信息，类型为 `BackendRouteMeta`：

```ts
export type BackendRouteMeta = {
  title: string;
  titleKey?: string;
  icon?: string;
  showInMenu?: boolean;
  order?: number;
};
```

字段含义：

- `title`：标题兜底文案。没有 `titleKey` 或国际化不可用时使用。
- `titleKey`：国际化 key。菜单生成时会优先通过 `t(titleKey)` 获取多语言标题。
- `icon`：菜单图标名称，需要在 `src/router/menu.tsx` 的 `iconMap` 中存在。
- `showInMenu`：是否展示在菜单中。不展示在菜单中不代表路由不可访问。
- `order`：排序值，数值越小越靠前。路由转换和菜单生成都会按该字段排序。

## 路由转换规则

`transformRoutes` 会递归处理 `mockRoutes`：

```tsx
export function transformRoutes(routes: BackendRouteItem[]): AppRouteObject[] {
  return routes
    .slice()
    .sort((a, b) => (a.meta?.order ?? 0) - (b.meta?.order ?? 0))
    .map((route) => {
      const appRoute: AppRouteObject = {
        path: route.path,
        meta: route.meta,
      };

      if (route.children?.length) {
        appRoute.children = transformRoutes(route.children);
      }

      if (route.component) {
        appRoute.element = lazyLoad(route.component);
      } else if (route.children?.length) {
        appRoute.element = <Outlet />;
      }

      if (route.redirect) {
        appRoute.children = [
          {
            index: true,
            element: <Navigate to={route.redirect} replace />,
          },
          ...(appRoute.children ?? []),
        ];
      }

      return appRoute;
    });
}
```

转换规则总结：

- 先按 `meta.order` 从小到大排序。
- 有 `children` 时递归转换子路由。
- 有 `component` 时懒加载对应页面组件。
- 没有 `component` 但有 `children` 时，父路由使用 `<Outlet />`。
- 有 `redirect` 时，会在子路由最前面插入一个 `index` 重定向路由。
- 如果 `component` 找不到对应页面模块，会渲染 `NotFound`。

## 菜单生成规则

`createMenuItems` 使用同一份 `mockRoutes` 生成 Ant Design 菜单：

```tsx
const menuItems = useMemo(
  () => createMenuItems(mockRoutes, t),
  [t, i18n.language],
);
```

生成规则：

- 只包含 `meta.showInMenu` 为 `true` 的路由。
- 按 `meta.order` 从小到大排序。
- 菜单 `key` 是拼接后的完整路径，例如 `/showcases/google-2d-tiles`。
- 点击菜单时通过 `navigate(key)` 跳转。
- 标题优先使用 `titleKey` 对应的国际化文案。
- 图标通过 `meta.icon` 从 `iconMap` 中读取。

如果新增了菜单图标，需要先在 `src/router/menu.tsx` 中导入图标并加入 `iconMap`：

```tsx
import { AppstoreOutlined } from "@ant-design/icons";

const iconMap = {
  AppstoreOutlined,
};
```

然后在路由配置中使用：

```ts
meta: {
  icon: "AppstoreOutlined",
}
```

## 新增一级页面路由

假设要新增 `/tools` 页面。

### 1. 新增页面组件

新增文件：

```text
src/pages/tools/index.tsx
```

示例：

```tsx
export default function ToolsPage() {
  return <div>工具页面</div>;
}
```

### 2. 修改路由配置

在 `src/router/mockRoutes.ts` 的 `mockRoutes` 数组中新增：

```ts
{
  path: "tools",
  name: "Tools",
  component: "tools",
  meta: {
    title: "工具",
    titleKey: "menu.tools",
    icon: "SettingOutlined",
    showInMenu: true,
    order: 5,
  },
}
```

### 3. 补充国际化文案

在 `src/i18n/locales/zh-CN.ts` 中补充：

```ts
menu: {
  tools: "工具",
}
```

在 `src/i18n/locales/en-US.ts` 中补充：

```ts
menu: {
  tools: "Tools",
}
```

注意保留现有 `menu` 结构，不要覆盖其他字段。

## 新增带子页面的分组路由

假设要新增一个 `/terrain` 分组，并包含 `/terrain/basic` 页面。

### 1. 新增页面组件

新增文件：

```text
src/pages/terrain/basic/index.tsx
```

示例：

```tsx
export default function TerrainBasicPage() {
  return <div>基础地形页面</div>;
}
```

### 2. 修改路由配置

在 `src/router/mockRoutes.ts` 中新增：

```ts
{
  path: "terrain",
  name: "Terrain",
  redirect: "/terrain/basic",
  meta: {
    title: "地形",
    titleKey: "menu.terrain.title",
    icon: "DashboardOutlined",
    showInMenu: true,
    order: 6,
  },
  children: [
    {
      path: "basic",
      name: "TerrainBasic",
      component: "terrain/basic",
      meta: {
        title: "基础地形",
        titleKey: "menu.terrain.basic",
        showInMenu: true,
        order: 1,
      },
    },
  ],
}
```

这里父路由没有 `component`，所以转换后会使用 `<Outlet />`。访问 `/terrain` 时会通过 `redirect` 跳转到 `/terrain/basic`。

### 3. 补充国际化文案

```ts
menu: {
  terrain: {
    title: "地形",
    basic: "基础地形",
  },
}
```

英文文案示例：

```ts
menu: {
  terrain: {
    title: "Terrain",
    basic: "Basic Terrain",
  },
}
```

## 常见注意事项

- 子路由 `path` 不要以 `/` 开头，`redirect` 建议使用完整路径。
- `component` 必须能对应到 `src/pages/${component}/index.tsx`。
- 需要显示在菜单中的路由必须设置 `meta.showInMenu: true`。
- `titleKey` 对应的国际化字段需要在所有语言文件中同步补充。
- `meta.icon` 只能使用 `iconMap` 中已经注册的图标名称。
- 父级分组路由如果没有自己的页面，建议配置 `redirect` 指向默认子页面。
- 新增或调整路由后，建议执行 `pnpm lint` 和 `pnpm build`，并手动验证页面跳转、菜单选中态和语言切换。
