# i18n 配置和使用

本文档总结当前项目中英文切换能力的推荐实现方式，包括依赖安装、目录结构、初始化配置、Provider 接入、组件中使用方式，以及路由 `titleKey` 的设计规范。

## 目标

项目需要支持：

- 侧边栏菜单根据当前语言切换中英文。
- 顶部 `Header` 中提供语言切换入口。
- 刷新页面后保留用户选择的语言。
- 路由配置只保存稳定的 `titleKey`，菜单文案由 i18n 统一管理。

## 安装依赖

推荐使用 `i18next` 和 `react-i18next`：

```bash
pnpm add i18next react-i18next
```

其中：

- `i18next` 负责国际化核心能力。
- `react-i18next` 负责在 React 组件中通过 Hook 使用翻译能力。

## 推荐目录结构

```txt
src/
  i18n/
    index.ts
    types.ts
    locales/
      zh-CN.ts
      en-US.ts
  app/
    providers/
      I18nProvider.tsx
  layouts/
    MainLayout/
      Header.tsx
      header-actions/
        LanguageSwitch.tsx
  router/
    mockRoutes.ts

```

职责说明：

- `src/i18n/index.ts`：初始化 i18n。
- `src/i18n/types.ts`：维护语言类型。
- `src/i18n/locales/zh-CN.ts`：中文文案。
- `src/i18n/locales/en-US.ts`：英文文案。
- `src/app/providers/I18nProvider.tsx`：应用级 i18n 初始化入口。
- `src/layouts/MainLayout/header-actions/LanguageSwitch.tsx`：语言切换按钮和下拉菜单。
- `src/router/mockRoutes.ts`：路由配置中维护 `titleKey`。
- `src/router/menu.tsx`：根据 `titleKey` 和 `t` 函数生成菜单文案。

## 语言类型

可以新增 `src/i18n/types.ts`：

```ts
export type AppLanguage = "zh-CN" | "en-US";
```

后续切换语言、读取语言时都使用这个类型，避免到处写散落的字符串。

## 初始化 i18n

`src/i18n/index.ts` 示例：

```ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";

import enUS from "./locales/en-US";
import zhCN from "./locales/zh-CN";
import type { AppLanguage } from "./types";

const LANGUAGE_STORAGE_KEY = "language";

const getDefaultLanguage = (): AppLanguage => {
  const language = localStorage.getItem(LANGUAGE_STORAGE_KEY);

  if (language === "zh-CN" || language === "en-US") {
    return language;
  }

  return "zh-CN";
};

i18n.use(initReactI18next).init({
  resources: {
    "zh-CN": {
      translation: zhCN,
    },
    "en-US": {
      translation: enUS,
    },
  },
  lng: getDefaultLanguage(),
  fallbackLng: "zh-CN",
  interpolation: {
    escapeValue: false,
  },
});

export { LANGUAGE_STORAGE_KEY };
export default i18n;
```

说明：

- `lng` 表示当前语言。
- `fallbackLng` 表示翻译缺失时的兜底语言。
- `escapeValue: false` 是 React 项目中的常见配置，因为 React 默认会处理 XSS 转义。
- 语言从 `localStorage` 读取，保证刷新后仍然保留用户选择。

## 接入 I18nProvider

`src/app/providers/I18nProvider.tsx`：

```tsx
import type { ReactNode } from "react";

import "@/i18n";

type I18nProviderProps = {
  children: ReactNode;
};

export function I18nProvider({ children }: I18nProviderProps) {
  return children;
}
```

`I18nProvider` 本身不需要渲染额外组件，核心作用是确保 `src/i18n/index.ts` 在应用启动时被执行。

然后在 `src/app/providers/AppProvider.tsx` 中接入：

```tsx
import { RouterProvider } from "react-router";

import router from "@/router";
import { I18nProvider } from "./I18nProvider";

export function AppProvider() {
  return (
    <I18nProvider>
      <RouterProvider router={router} />
    </I18nProvider>
  );
}
```

后期如果还有主题、Ant Design 全局配置，可以继续组合：

```tsx
export function AppProvider() {
  return (
    <I18nProvider>
      <ThemeProvider>
        <AntdProvider>
          <RouterProvider router={router} />
        </AntdProvider>
      </ThemeProvider>
    </I18nProvider>
  );
}
```

## 路由 titleKey 设计

路由配置中不要直接依赖菜单中文文案，而是保留 `titleKey`：

```ts
{
  path: "getting-started",
  name: "GettingStarted",
  redirect: "/getting-started/hello-world",
  meta: {
    title: "快速开始",
    titleKey: "menu.gettingStarted.title",
    icon: "UserOutlined",
    showInMenu: true,
    order: 2,
  },
  children: [
    {
      path: "hello-world",
      name: "HelloWorld",
      component: "getting-started/hello-world",
      meta: {
        title: "Hello World",
        titleKey: "menu.gettingStarted.helloWorld",
        showInMenu: true,
        order: 1,
      },
    },
    {
      path: "resolution-scale",
      name: "ResolutionScale",
      component: "getting-started/resolution-scale",
      meta: {
        title: "分辨率缩放",
        titleKey: "menu.gettingStarted.resolutionScale",
        showInMenu: true,
        order: 2,
      },
    },
  ],
}
```

推荐规则：

- 父级菜单使用 `menu.xxx.title`。
- 子级菜单使用 `menu.xxx.yyy`。
- `meta.title` 只作为兜底文案。
- `meta.titleKey` 是菜单、面包屑、页面标题的主要文案来源。

不要这样设计：

```ts
titleKey: "menu.gettingStarted";
```

同时又写：

```ts
titleKey: "menu.gettingStarted.helloWorld";
```

因为 `menu.gettingStarted` 不能既是字符串，又是对象。推荐统一为：

```txt
menu.gettingStarted.title
menu.gettingStarted.helloWorld
menu.gettingStarted.resolutionScale
```

## 菜单生成函数

`src/router/menu.tsx` 不要直接调用 `useTranslation`，它应该保持为纯函数。由组件调用 `useTranslation` 后，把 `t` 函数传进来。

示例：

```tsx
import { createElement } from "react";
import {
  DashboardOutlined,
  HomeOutlined,
  SettingOutlined,
  UserOutlined,
} from "@ant-design/icons";
import type { MenuProps } from "antd";
import type { TFunction } from "i18next";

import type { BackendRouteItem } from "./types";

type MenuItem = Required<MenuProps>["items"][number];

const iconMap = {
  HomeOutlined,
  UserOutlined,
  DashboardOutlined,
  SettingOutlined,
};

function joinPath(parentPath: string, path: string) {
  const normalizedParent = parentPath.endsWith("/")
    ? parentPath.slice(0, -1)
    : parentPath;

  return `${normalizedParent}/${path}`.replace(/\/+/g, "/");
}

export function createMenuItems(
  routes: BackendRouteItem[],
  t: TFunction,
  parentPath = "",
): MenuItem[] {
  return routes
    .filter((route) => route.meta?.showInMenu)
    .sort((a, b) => (a.meta?.order ?? 0) - (b.meta?.order ?? 0))
    .map((route) => {
      const fullPath = joinPath(parentPath, route.path);
      const iconName = route.meta?.icon;
      const Icon = iconName
        ? iconMap[iconName as keyof typeof iconMap]
        : undefined;

      const label = route.meta?.titleKey
        ? t(route.meta.titleKey)
        : route.meta?.title;

      const children = route.children
        ? createMenuItems(route.children, t, fullPath)
        : undefined;

      return {
        key: fullPath,
        icon: Icon ? createElement(Icon) : undefined,
        label,
        children: children?.length ? children : undefined,
      };
    });
}
```

## Sidebar 中使用

`src/layouts/MainLayout/Sidebar.tsx` 示例：

```tsx
import { useMemo } from "react";
import { Layout, Menu } from "antd";
import { useLocation, useNavigate } from "react-router";
import { useTranslation } from "react-i18next";
import type { MenuProps } from "antd";

import { mockRoutes } from "@/router/mockRoutes";
import { createMenuItems } from "@/router/menu";

const { Sider } = Layout;

export function MainLayoutSidebar() {
  const location = useLocation();
  const navigate = useNavigate();
  const { t, i18n } = useTranslation();

  const menuItems = useMemo(
    () => createMenuItems(mockRoutes, t),
    [t, i18n.language],
  );

  const onMenuClick: MenuProps["onClick"] = ({ key }) => {
    navigate(key);
  };

  return (
    <Sider breakpoint="lg" collapsedWidth="0" className="hidden lg:block">
      <Menu
        theme="dark"
        mode="inline"
        selectedKeys={[location.pathname]}
        items={menuItems}
        onClick={onMenuClick}
      />
    </Sider>
  );
}
```

注意：

- `useTranslation` 放在组件里。
- `createMenuItems` 不直接调用 Hook。
- `selectedKeys` 应该使用当前 `location.pathname`，不要使用只初始化一次的 `defaultSelectedKeys`。
- `i18n.language` 放入 `useMemo` 依赖，语言变化后菜单会重新生成。

## 移动端菜单中使用

`src/layouts/MainLayout/MobileDrawer.tsx` 的用法和 `Sidebar` 基本一致：

```tsx
const { t, i18n } = useTranslation();

const menuItems = useMemo(
  () => createMenuItems(mockRoutes, t),
  [t, i18n.language],

```

移动端菜单点击后建议关闭抽屉：

```tsx
const onMenuClick: MenuProps["onClick"] = ({ key }) => {
  navigate(key);
  onClose();
};
```

## Header 语言切换组件

如果 `Header` 中会放多个图标和操作入口，推荐把语言切换封装为单独组件：

```txt
src/layouts/MainLayout/
  Header.tsx
  header-actions/
    LanguageSwitch.tsx
```

`src/layouts/MainLayout/header-actions/LanguageSwitch.tsx` 示例：

```tsx
import { TranslationOutlined } from "@ant-design/icons";
import { Button, Dropdown, Tooltip } from "antd";
import { useTranslation } from "react-i18next";
import type { MenuProps } from "antd";

import { LANGUAGE_STORAGE_KEY } from "@/i18n";
import type { AppLanguage } from "@/i18n/types";

const languageItems: MenuProps["items"] = [
  {
    key: "zh-CN",
    label: "中文",
  },
  {
    key: "en-US",
    label: "English",
  },
];

export function LanguageSwitch() {
  const { t, i18n } = useTranslation();

  const onLanguageClick: MenuProps["onClick"] = ({ key }) => {
    const language = key as AppLanguage;

    i18n.changeLanguage(language);
    localStorage.setItem(LANGUAGE_STORAGE_KEY, language);
  };

  return (
    <Tooltip title={t("header.switchLanguage")}>
      <Dropdown
        menu={{
          items: languageItems,
          selectable: true,
          selectedKeys: [i18n.language],
          onClick: onLanguageClick,
        }}
        trigger={["click"]}
      >
        <Button
          type="text"
          icon={<TranslationOutlined />}
          aria-label={t("header.switchLanguage")}
        />
      </Dropdown>
    </Tooltip>
  );
}
```

`Header.tsx` 中只负责摆放：

```tsx
import { MenuOutlined } from "@ant-design/icons";
import { Button, Layout } from "antd";

import { LanguageSwitch } from "./header-actions/LanguageSwitch";

const { Header } = Layout;

type MainLayoutHeaderProps = {
  onMenuClick: () => void;
};

export function MainLayoutHeader({ onMenuClick }: MainLayoutHeaderProps) {
  return (
    <Header className="flex h-14 items-center justify-between bg-white px-4">
      <Button
        type="text"
        icon={<MenuOutlined />}
        className="lg:hidden"
        onClick={onMenuClick}
      />

      <div className="ml-auto flex items-center gap-2">
        <LanguageSwitch />
      </div>
    </Header>
  );
}
```

## useTranslation 放在哪里

推荐放在真正需要读取或切换文案的 React 组件中：

- `Sidebar.tsx`：菜单需要翻译。
- `MobileDrawer.tsx`：移动端菜单需要翻译。
- `LanguageSwitch.tsx`：按钮提示和语言切换需要翻译。
- 普通页面组件：页面内文案需要翻译。

不推荐放在：

- `mockRoutes.ts`：这是纯配置文件，不是 React 组件。
- `transform.tsx`：这是路由转换函数。
- `menu.tsx` 顶层：这是菜单生成工具函数。

原因是 React Hook 只能在 React 组件或自定义 Hook 中调用。配置文件和普通工具函数不应该直接调用 Hook。

## 是否需要封装 useLanguage

当前阶段可以不封装 Hook，直接在 `LanguageSwitch` 中写切换逻辑即可。

当语言切换逻辑被多个地方复用时，再新增：

```txt
src/i18n/useLanguage.ts
```

示例：

```ts
import { useTranslation } from "react-i18next";

import { LANGUAGE_STORAGE_KEY } from "@/i18n";
import type { AppLanguage } from "./types";

export function useLanguage() {
  const { i18n } = useTranslation();

  const changeLanguage = (language: AppLanguage) => {
    i18n.changeLanguage(language);
    localStorage.setItem(LANGUAGE_STORAGE_KEY, language);
  };

  return {
    language: i18n.language as AppLanguage,
    changeLanguage,
  };
}
```

## 实施顺序

推荐按以下顺序实现：

1. 安装 `i18next` 和 `react-i18next`。
2. 新增 `src/i18n/types.ts`。
3. 完善 `src/i18n/locales/zh-CN.ts` 和 `src/i18n/locales/en-US.ts`。
4. 完善 `src/i18n/index.ts`。
5. 在 `AppProvider` 中接入 `I18nProvider`。
6. 统一调整路由中的 `meta.titleKey`，父级菜单使用 `menu.xxx.title`。
7. 修改 `createMenuItems`，通过传入的 `t` 函数生成菜单标题。
8. 在 `Sidebar` 和 `MobileDrawer` 中使用 `useTranslation`。
9. 新增 `LanguageSwitch` 组件。
10. 在 `Header` 中放入 `LanguageSwitch`。
11. 执行构建验证。

验证命令：

```bash
pnpm.cmd build
```

需要验证的结果：

- 点击顶部语言图标后可以选择中文或英文。
- 侧边栏菜单文案随语言切换。
- 移动端抽屉菜单文案随语言切换。
- 刷新页面后保留上次选择的语言。
- 路由访问和菜单跳转不受语言切换影响。
