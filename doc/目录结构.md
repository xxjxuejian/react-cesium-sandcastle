# 目录结构说明

## src/app 的职责

`src/app` 用来放应用级装配代码，也就是整个 React 应用启动时需要统一包起来的内容。

它的核心职责是：

- 组织应用入口组件。
- 统一组合全局 Provider。
- 放置全局初始化逻辑。
- 管理应用级配置的接入位置，例如路由、主题、国际化、Ant Design 全局配置等。

可以把 `src/app` 理解为：

```txt
src/app = 应用外壳 + 全局 Provider + 全局初始化
```

它不应该承载具体业务页面，也不应该放 Cesium 示例逻辑、普通工具函数或业务组件。

## 推荐目录结构

当前阶段可以先使用较轻量的结构：

```txt
src/
  app/
    App.tsx
    providers/
      AppProvider.tsx
```

后期加入主题切换、国际化、Ant Design 全局配置后，可以扩展为：

```txt
src/
  app/
    App.tsx
    providers/
      AppProvider.tsx
      AntdProvider.tsx
      ThemeProvider.tsx
      I18nProvider.tsx
```

## 当前阶段示例

### src/app/App.tsx

`App.tsx` 只负责加载应用级 Provider，不直接写路由细节、页面内容或业务逻辑。

```tsx
import { AppProvider } from "./providers/AppProvider";

export default function App() {
  return <AppProvider />;
}
```

### src/app/providers/AppProvider.tsx

`AppProvider` 是全局 Provider 的组合入口。当前阶段只需要接入 React Router。

```tsx
import { RouterProvider } from "react-router";

import router from "@/router";

export function AppProvider() {
  return <RouterProvider router={router} />;
}
```

### src/main.tsx

`main.tsx` 只负责挂载 React 根节点，并引入全局样式。

```tsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";

import App from "@/app/App";
import "@/index.css";

createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <App />
  </StrictMode>,
);
```

## 后期扩展示例

当项目需要主题切换、国际化、Ant Design 全局配置时，可以把这些能力继续收敛到 `src/app/providers` 中。

### src/app/providers/AppProvider.tsx

```tsx
import { RouterProvider } from "react-router";

import router from "@/router";
import { AntdProvider } from "./AntdProvider";
import { I18nProvider } from "./I18nProvider";
import { ThemeProvider } from "./ThemeProvider";

export function AppProvider() {
  return (
    <I18nProvider>
      <ThemeProvider>
        <AntdProvider>
          <RouterProvider router={router} />
        </AntdProvider>
      </ThemeProvider>
    </I18nProvider>
  );
}
```

### src/app/providers/AntdProvider.tsx

`AntdProvider` 负责 Ant Design 的全局配置，例如主题算法、主色、圆角、组件默认配置等。

```tsx
import type { ReactNode } from "react";
import { ConfigProvider, theme } from "antd";

type AntdProviderProps = {
  children: ReactNode;
};

export function AntdProvider({ children }: AntdProviderProps) {
  return (
    <ConfigProvider
      theme={{
        algorithm: theme.defaultAlgorithm,
        token: {
          colorPrimary: "#1677ff",
          borderRadius: 6,
        },
      }}
    >
      {children}
    </ConfigProvider>
  );
}
```

### src/app/providers/ThemeProvider.tsx

`ThemeProvider` 负责维护全局主题状态，例如 `light` 和 `dark`。后期可以在这里同步 `localStorage` 和 `document.documentElement.dataset.theme`。

```tsx
import type { ReactNode } from "react";

type ThemeProviderProps = {
  children: ReactNode;
};

export function ThemeProvider({ children }: ThemeProviderProps) {
  return children;
}
```

后期主题逻辑可以演进为：

```tsx
import { createContext, useEffect, useMemo, useState } from "react";
import type { ReactNode } from "react";

type ThemeMode = "light" | "dark";

type ThemeContextValue = {
  themeMode: ThemeMode;
  setThemeMode: (themeMode: ThemeMode) => void;
};

export const ThemeContext = createContext<ThemeContextValue | null>(null);

type ThemeProviderProps = {
  children: ReactNode;
};

export function ThemeProvider({ children }: ThemeProviderProps) {
  const [themeMode, setThemeMode] = useState<ThemeMode>("light");

  useEffect(() => {
    document.documentElement.dataset.theme = themeMode;
    localStorage.setItem("themeMode", themeMode);
  }, [themeMode]);

  const value = useMemo(
    () => ({
      themeMode,
      setThemeMode,
    }),
    [themeMode],
  );

  return (
    <ThemeContext.Provider value={value}>{children}</ThemeContext.Provider>
  );
}
```

### src/app/providers/I18nProvider.tsx

`I18nProvider` 负责接入国际化初始化逻辑。如果使用 `react-i18next`，通常可以在这里引入 `src/i18n` 初始化文件。

```tsx
import type { ReactNode } from "react";

import "@/i18n";

type I18nProviderProps = {
  children: ReactNode;
};

export function I18nProvider({ children }: I18nProviderProps) {
  return children;
}
```

## 不建议放入 src/app 的内容

以下内容不建议放进 `src/app`：

- 页面组件。
- 业务组件。
- Cesium Viewer 初始化和示例逻辑。
- 请求接口。
- 普通工具函数。
- 普通业务 hooks。
- 路由页面配置。
- 全局通用 UI 组件。

这些内容更适合放在：

```txt
src/pages
src/features
src/components
src/hooks
src/api
src/router
src/utils
```

## 推荐边界

`src/app` 只负责把应用装起来，不负责具体业务。

建议保持这样的依赖方向：

```txt
main.tsx
  -> src/app/App.tsx
    -> src/app/providers/AppProvider.tsx
      -> src/router
      -> 主题 Provider
      -> 国际化 Provider
      -> Ant Design Provider
```

业务页面、布局、组件不应该反向依赖 `src/app`。
